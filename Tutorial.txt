## R-PAKET ENTWICKLUNG - CHEATSHEET

### INITIALES SETUP (einmalig pro Paket)

```r
# 1. Paket erstellen
usethis::create_package("pfad/zum/paket/paketname")

# 2. DESCRIPTION-Datei manuell editieren
# - Title, Authors@R, Description anpassen

# 3. Lizenz setzen
usethis::use_mit_license()

# 4. Optional: Git initialisieren
usethis::use_git()
```

### NEUE FUNKTIONEN HINZUFÜGEN (wiederholt)

```r
# 1. Neue R-Datei erstellen
usethis::use_r("dateiname")

# 2. Funktion mit roxygen2-Dokumentation schreiben
# (Siehe Syntax-Referenz unten)

# 3. Dokumentation generieren
devtools::document()

# 4. Paket laden und testen
devtools::load_all()

# 5. Funktion testen in der Konsole
funktionsname(argumente)

# 6. Hilfedatei prüfen
?funktionsname
```

### ROXYGEN2-SYNTAX-REFERENZ

```r
#' Kurzer Titel in einer Zeile
#'
#' Optionale längere Beschreibung kann hier stehen.
#' Kann mehrere Zeilen umfassen.
#'
#' @param param1 Beschreibung des ersten Parameters
#' @param param2 Beschreibung des zweiten Parameters
#' @return Beschreibung was zurückgegeben wird
#' @export
#' @examples
#' funktionsname(wert1, wert2)
#' funktionsname(100, 50)
funktionsname <- function(param1, param2) {
  # Code hier
  ergebnis <- param1 / param2
  ergebnis
}
```

### WICHTIGE REGELN

- **Immer `#'` für roxygen2**, nicht `#` oder `###`
- **Jeder Parameter braucht eigene `@param`-Zeile**
- **`@examples` muss ausführbarer R-Code sein**, keine Formeln
- **NAMESPACE und .Rd-Dateien NIE händisch editieren** - nur über roxygen2
- **Projekt muss geladen sein** - oben rechts muss Paketname stehen
- **Dezimaltrennzeichen ist Punkt** (1.5), nicht Komma (1,5)

### STANDARD-WORKFLOW

1. `usethis::use_r("dateiname")` → Datei erstellen
2. Code + Dokumentation schreiben
3. `devtools::document()` → Dokumentation generieren
4. `devtools::load_all()` → Paket laden
5. Funktion testen
6. Bei Änderungen: Zurück zu Schritt 2

### HÄUFIGE FEHLER

- Projekt nicht geladen → `devtools::load_all()` findet Paket nicht
- `###'` statt `#'` → roxygen2 ignoriert Dokumentation
- `@examples` mit Formeln statt Code → Syntax-Error beim Testen
- Falsche Mathematik (multiplizieren statt dividieren)
- NAMESPACE händisch editiert → wird bei `document()` überschrieben
